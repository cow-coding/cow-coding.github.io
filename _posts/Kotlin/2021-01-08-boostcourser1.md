# Kotlin(코틀린) 이란?
## Kotlin이 유명해지게 된 배경
코틀린은 요즈음 가장 핫한 언어라고 할 수 있다. 코틀린에 대해 말하기 전에 일단 자바(Java)에 대해서 말하지 않을 수가 없다. 자바는 국내뿐 아니라 전 세계적으로 오랜 시간 핵심 개발 언어로 자리 잡고 있는 언어였다. 서버 개발에 사용되는 스프링 프레임워크, 안드로이드, 자바 프로그래밍 등등 안 쓰이는 분야가 없다고 할 정도로 모든 분야에 사용이 되었다.  
하지만 자바에서 발생하는 일부 문제와 자꾸 저작권 트러블을 일으키는 오라클 때문에 구글은 안드로이드 공식 언어로 JetBrains에서 공개한 언어인 코틀린을 추가했고 이와 동시에 코틀린에 대한 관심이 급증했다.  
코틀린이 새롭게 나타난 언어지만 이렇게 빠르게 자리를 잡을 수 있던 이유는 다름 아닌 자바와의 100% 호환성을 가진 언어라는 것이다. 기존의 자바코드에 코틀린을 얹어도 충분히 문제없이 돌아간다는 것이다. 즉 언어를 다 갈아엎는다면 엄청난 유지보수 비용이 들어가게 되는데, 자바와의 호환성이 100%이기 때문에 더 간결하고 좋은 언어를 굳이 거를 필요가 없게 된 것이다.  
구글은 안드로이드의 공식문서 예제도 Kotlin이 Java보다 먼저 나오게 되었고 국내에서는 카카오가 카카오톡 메시징 서버에 Kotlin을 테스트로 사용해 본 결과, 코드량이 비약적으로 감소하고 생산성이 대폭 향상되어서 코틀린으로 언어를 변경하는 작업을 한다고 한다. [참고](https://byline.network/2018/09/5-20/)  

## Kotlin의 특징
1. 자바 개발툴로 유명한 IntelliJ를 만든 JetBrains에서 개발하고 보급하는 언어이다. 이런 이유로 자바와의 호환성이 좋았던 것 같다.
2. 코드 자체의 간결성을 굉장히 높여주었다. 진짜 간단한 Hello World 예제를 비교해보자.
    < Java 코드 >
    ```java
    class Main {
      public static void main(String[] args) {
        System.out.println("Hello, world!");
      }
    }
    ```  
   < Kotlin 코드 >
   ```kotlin
   fun main() {
    println("Hello World!")
   }
   ```
   요 정도 코드로는 비교가 좀 어렵지만 이런 간단한 것만 봐도 코드 길이가 다르다.
3. Nullable 지원으로인해 코드 안정성이 높다.
   프로그래밍을 하면서 어떤 프로그램을 만들다보면 Null Pointer Exception에러가 굉~~~장히 자주 일어난다는 것을 알 것이다. 특히 자바에서 NPE는 기본 세팅 오류 수준으로 나타나는 오류이다. 하지만 여기서 변수를 선언할 때 아래와 같이 선언을 함으로써 Nullable을 지원해서 NPE 오류에서 자유로워진다.
   ```kotlin
   return b?.length // b가 null이면 null을 리턴
   ```
4. 그 외에 세미콜론이 없어도 괜찮다. 물론 넣어도 괜찮다. 그리고 더 많은 것들이 있다. 자세한 것은 공식문서를 참고해보기를 추천한다.


# 개발 환경
## JDK 설치
우선 JVM이 필요하기 때문에 JDK를 설치해야한다. 나는 맥북이라 맥북에 맞춰서 개발환경을 설정해주겠다.  
개인적으로 공부를 하는 환경에서는 OpenJDK를 써보자.  
1. [OpenJDK](https://openjdk.java.net)로 들어가서 Mac용 JDK를 다운받자. JDK 8, 11이 LTS라서 가장 괜찮다. 보통 8을 많이 사용한다. 8로 다운받자.
2. 다운을 받고 압축을 해제하여 나오는 폴더를 /Library/Java/JavaVirtualMachines/에 넣어주면된다. 폴더가 없으면 만들어도 괜찮다.
3. 확인을 위해서 터미널에 아래와 같이 나오면 제대로 설치가 된 것이다.  

  ![](/image/boostcourse/jdk8.png)

## 자바 환경변수 설정
맥북은 환경변수 설정이 조금 번거롭다. 잘 따라오면 큰 문제없이 잘 해결될 것이다.

터미널 창을 키고 `sudo vim ~/.bash_profile`을 작성하면 아래와 같은 창이 나타난다.  

![env](/image/boostcourse/env.png)

나는 이미 설정을 했기 때문에 JAVA_HOME과 PATH가 적혀있지만 아마 처음 설정하는 사람은 없을 것이다. 저기 적혀있는 것처럼 아까 저장한 JDK의 이름에 맞춰서 적어주자.  
적은 뒤 ESC :wq를 하면 저장이되고 다시 콘솔창으로 나가진다.  
그 후 `source ~/.bash_profile`을 해주면 환경변수 설정이 끝난다.

## IntelliJ 설치
[IntelliJ 링크](https://www.jetbrains.com/idea/download)로 들어가면 인텔리제이를 설치할 수 있다. 만약 대학생이라면 JetBrains 학생 인증을 활용하자. 정말 좋다. 아니라면 Community로 설치하자. 설치하고 나오는 테마를 나는 드라큘라 테마로 했다. 이유는 따로 없고 밝은 IDE들을 눈이 아파서 별로 안 좋아한다. 후에 나오는 tools와 plugin은 따로 설치 안해도된다. 다 스킵하자.
