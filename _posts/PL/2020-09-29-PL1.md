# 프로그래밍 언어란?

프로그래밍 언어는 우리가 흔히 접하는 자바, 어셈블리어, C++과 같은 컴퓨터를 동작시키기 위해 만든 인공적인 언어를 말한다.  
일반적으로 저급언어, 고급언어라고 구분을 짓는데, 저급언어에는 대표적으로 어셈블리어가 있고 고급언어는 우리가 잘 아는 C, C++, JAVA와 같은 대부분의 언어가 있다.

## 저급언어

- 어셈블리어 (assembly_x86)

```nasm
mov ax, X
add ax, Y
mov Z, ax
```

~~마크다운 코드블럭도 지원 안하는 유명한 저급언어 어셈블리어다.~~  
안하는 줄 알았는데 해준다. 역시 킹갓 syntax highlight다.
심지어 어셈블리어는 마이크로프로세서가 달라지면 같은 명령어도 다른 단어를 사용한다.  
위에 적은 명령어는 Intel x86계열 CPU를 기준으로 처리한 것이다. 일부 다른 CPU에서는 `mov`를 `load`로 쓰기도 한다.
이미 여기서부터 언어로서 이식성이 상당히 떨어지는 것을 알 수 있다.  
대신 장점이라면 직접적으로 하드웨어를 제어할 수 있다는 점이 있다.

## 고급언어

- 자바 (JAVA)

```java
public class Test {
  public static void main(String[] args) {
    int Z;
    int X;
    int Y;
    Z = X + Y;
  }
}
```

한국에서는 제일 핫한 언어? 중 하나인 자바로 작성한 코드다. 사실 이 정도 구문은 자바든, C든, C++이든 비슷하게 적을 것이다.  
위의 코드는 사실 내부 동작과정은 어셈블리어의 형태로 동작한다. 결국 프로그래밍 언어의 세세한 부분을 알기 위해서는 저급언어를 파악할 필요가 있기는 하다.  
물론 본인이 직접 언어를 만들던가 수정하는 것이 아니라면 크게 필요하진 않다.

# 프로그래밍 언어 설계 원칙

프로그래밍 언어론과 관련된 서적들을 찾아보면 단어나 워딩은 차이가 있더라도 실제로 내용을 보면 비슷비슷한 말을 하고 있다. 설계 원칙이 꽤 여러가지가 있는데 주요 몇가지를 알아보자.

## 효율성

프로그래밍 언어를 설계하고 만들 때 효율적인 부분을 고려하는 것은 당연하다고 볼 수 있다.  

- 목적 코드의 효율성 : 번역기가 효율**적인** 실행 코드를 만들어야 함
- 번역의 효율성 : 번역기가 효율**적으로** 실행 코드를 만들어야 함

효율성만 따졌을 때 유명한 언어 중에 가장 뛰어난 언어는 아마 파이썬(Python)이지 않을까 싶다.

```py
print("hello python!")
```

파이썬은 자바, C++에 비해서 특정한 것들을 사용할 때 필요한 함수나 라이브러리가 단순하게 잘 구조화되어 있다.  
그래서 요즘 파이썬이 핫하고 많은 사람들이 배우는 데에 어려움을 느끼지 않는 것이다. 물론 다른 부분으로 봤을 때도 뛰어난 언어인지는 더 확인을 해봐야한다.

## 일반성

- 특별한 경우를 피하고 밀접하게 관련된 개념들을 일반적인 것으로 결합하는 성질이다.

이게 개념으로만 보면 뭔 소린지 잘 모른다.  
간단하게 말하면 비슷한 개념은 비슷한 행동을 할 수 있다는 것이다. 우리가 C++이나 자바에서 배열에 대입연산자인 (=)을 못 쓴다던가, call by reference만 가능하다던가 하지 않고 어떤 대상에서든 가능한 경우를 보여준다.  
(물론 자바는 포인터가 없어서 참조를 활용하게 하는 것으로 부족한 일반성을 올려줬다.)

## 획일성

- 비슷한 것은 비슷한게 보이고 비슷한 의미를 가져야한다. 다르면 달라야 한다.

획일성은 말 그대로 비슷하게 생긴 애들은 비슷한 역할을 해야한다는 것이다.  
획일성이 부족한 대표적인 예는 C++에서 나타난다.

```cpp
class A {...}; // 세미콜론 있음

void f() {...} // 세미콜론 없음
```

C++에서 클래스 선언과 함수 선언은 매우 비슷한 형태로 나타난다. 하지만 둘의 역할은 비슷하지 않다. 이런 것이 획일성이 부족한 경우이다.

## 간결성

- 언어는 복잡하지 않고 간결해야 한다.

이번 수업을 들으면서 그냥 맛만 본 언어 중에 Prolog라는 언어가 있는데, 논리 명제를 검증해주는 프로그래밍 언어이다.  
이 언어는 생각보다 간결한 작성방식을 띄고 있다. 간결성이 높은 언어로는 Go-lang도 간결하다고 볼 수 있다.  
Go-lonag은 실제로 구성 자체가 그렇게 많지 않다.

## 확장성

- 사용자가 언어에 새로운 기능을 추가할 수 있도록 하는 성질

우리가 사용하는 많은 프로그래밍 언어들은 확장성들이 좋다. 실제로 자바는 벌써 13까지 나왔고 파이썬은 3.9버전이 pre-release인 것으로 알고 있다. C++은 20까지 발전되어 왔다.  

## 기계 독립성

- 언어는 기계에 의존적이지 않고 독립적이어야 한다.

기계 독립성에 제약이 가장 심한 언어가 바로 위에서 예로 든 어셈블리어이다.  
마이크로 프로세서의 종류에 따라 명령어가 변경된다니... 이식성을 저해하는 요소가 된다.
JAVA가 흥행한 이유는 기계 독립성이 매우 뛰어난 것도 한 몫을 했다. J
AVA는 기본적으로 JVM(Java Virtual Machine)을 통해서 구동되기 때문에 어떤 컴퓨터, OS에서도 동작할 수 있다.
